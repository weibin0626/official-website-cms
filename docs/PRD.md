## 一、产品概述

### 1.1 产品定位

面向教育集团/多法人机构的企业级内容管理与官网发布平台。系统需满足等保三级安全合规要求，支持多站点独立运营、多级审核工作流、智能内容管理，以及全面的运维运营支撑能力。

### 1.2 目标用户

| 用户类型 | 描述 |
|---------|------|
| 集团管理层 | 管理多个法人站点，关注全局数据与安全合规 |
| 站点管理员 | 单一站点运营负责人，管理本站点内容与人员 |
| 内容编辑 | 日常内容采编人员，撰写和提交内容 |
| 审核员 | 负责内容审核、把关内容质量 |
| 普通用户/访客 | 前台网站浏览者 |

### 1.3 核心价值

1. **安全合规**：满足 GB/T 22239-2019 第三级要求，通过专业安全检测
2. **多法人独立运营**：每个法人站点数据、权限、模板完全隔离，互不干扰
3. **高效内容管理**：从采编发到审核退稿的完整工作流，支持富文本编辑、多模型内容管理

### 1.4 原始需求复述

基于《官网.docx》需求文档，当前v1.0.0为纯前端原型（localStorage存储），验收完成率仅22%。需构建包含后端服务、数据库、安全防护、25项技术支撑能力的完整企业级CMS平台。核心需求涵盖：等保三级安全合规、内容管理系统、绩效统计报表、组织架构管理、统一认证与SSO、模板引擎、RBAC权限、消息服务、智能搜索、文件管理、内容检查、静态化发布等。

## 二、用户画像与角色

### 2.1 超级管理员（Super Admin）

- **职责**：管理所有站点、全局配置、用户体系、安全策略
- **典型场景**：新建法人站点、分配站点管理员、查看全局运维数据、配置安全策略
- **权限范围**：跨站点全局权限，无任何限制

### 2.2 站点管理员（Site Admin）

- **职责**：管理本站点内容、人员、模板、配置
- **典型场景**：管理栏目结构、分配编辑/审核人员、配置站点风格、查看站点统计
- **权限范围**：仅限所属站点，不可跨站点操作

### 2.3 内容编辑（Editor）

- **职责**：内容采编、提交审核、管理个人素材
- **典型场景**：撰写文章、上传图片/视频、提交审核、查看退稿原因并修改
- **权限范围**：仅可操作自己创建或被授权的内容

### 2.4 审核员（Reviewer）

- **职责**：审核内容、退稿、标记问题内容
- **典型场景**：审核待发布文章、退回不合格稿件、标记敏感内容
- **权限范围**：可审核本站点内所有待审内容

### 2.5 普通用户/访客（Visitor）

- **职责**：浏览前台内容、搜索、投稿
- **典型场景**：浏览新闻、搜索信息、提交投稿
- **权限范围**：仅前台浏览与投稿功能

## 三、功能需求池

> **优先级定义**：
> - **P0**：MVP必须上线，系统不可用若无此功能
> - **P1**：首批交付必须实现，影响核心业务流程
> - **P2**：增强功能，提升效率与体验
> - **P3**：锦上添花，可延后迭代

| 编号 | 功能模块 | 需求描述 | 验收标准 |
|------|---------|---------|---------|
| P0-01 | 后端服务搭建 | Node.js + Express 后端，PostgreSQL 数据库，RESTful API | 服务可启动，API可访问，数据库连接正常 |

## 四、多法人架构设计说明

### 4.1 站点隔离

```
┌─────────────────────────────────────────────┐
│                 CMS 平台（全局）                │
├─────────────┬──────────────┬─────────────────┤
│  站点 A     │   站点 B     │    站点 C        │
│ (春阳教育)  │  (德育中学)   │  (明德职校)      │
│             │              │                 │
│ · 独立栏目  │  · 独立栏目   │  · 独立栏目      │
│ · 独立内容  │  · 独立内容   │  · 独立内容      │
│ · 独立模板  │  · 独立模板   │  · 独立模板      │
│ · 独立用户  │  · 独立用户   │  · 独立用户      │
│ · 独立配置  │  · 独立配置   │  · 独立配置      │
└─────────────┴──────────────┴─────────────────┘
```

### 4.2 数据隔离策略

| 隔离维度 | 策略 | 说明 |
|---------|------|------|
| 内容数据 | 共享数据库，site_id字段隔离 | 文章、栏目、轮播图等所有内容均携带site_id，查询时强制过滤 |
| 用户数据 | 全局用户表 + 站点关联 | 用户全局唯一，通过site_users关联表映射到站点角色 |
| 文件资源 | 按站点目录隔离 | /uploads/{site_id}/结构存储，文件元数据携带site_id |
| 模板资源 | 按站点目录隔离 | /templates/{site_id}/结构存储 |
| 配置数据 | 每站点独立配置表 | site_configs表以site_id为主键 |
| 日志数据 | 共享日志表，site_id字段隔离 | 操作日志携带site_id，站点管理员仅可查本站日志 |

### 4.3 权限隔离策略

- **超级管理员**：可切换和操作所有站点，无site_id限制
- **站点管理员/编辑/审核员**：登录后绑定当前站点，所有操作自动注入site_id过滤
- **站点切换**：用户切换站点后，重新加载该站点的菜单、数据和权限
- **跨站点操作**：仅限内容推送(P2-05)，需接收站点审核

### 5.3 节点权限（栏目级权限控制）

- 每个栏目（节点）可独立配置哪些角色/用户有操作权限
- 支持字段级权限（如某用户只能编辑文章标题和正文，不能修改发布状态）
- 权限继承：子栏目默认继承父栏目权限，可覆盖

### 5.4 权限数据模型

```
User ←→ SiteUser ←→ Site
                 ↕
              Role ←→ Permission
                       ↕
                    Node (可选，栏目级)
```

- `User`: 全局用户
- `SiteUser`: 用户-站点-角色关联
- `Role`: 角色定义
- `Permission`: 权限项（资源+操作）
- `NodePermission`: 节点级权限覆盖

## 六、非功能需求

### 6.1 安全合规（等保三级 GB/T 22239-2019）

| 需求项 | 具体要求 | 优先级 |
|--------|---------|--------|
| 身份鉴别 | 密码复杂度(8位+大小写+数字+特殊字符)、登录失败锁定、会话超时 | P0 |
| 访问控制 | RBAC模型、最小权限原则、越权访问拦截 | P0 |
| 安全通信 | HTTPS强制、敏感数据传输加密 | P0 |
| 安全审计 | 操作日志(180天留存)、登录日志、异常行为告警 | P0 |
| 入侵防范 | CC攻击防护、SQL注入防护、XSS/CSRF防护、上传文件类型限制 | P0 |
| 数据完整性 | 数据库定期备份、操作事务一致性 | P1 |

### 7.1 后台管理界面

#### 7.1.1 整体布局

```
┌──────────────────────────────────────────────┐
│  Logo  │  站点切换器  │  🔔通知  │  用户头像 ▼ │  ← 顶部栏
├────┬───────────────────────────────────────────┤
│    │                                           │

│ ·用│                                           │
│ ·设│                                           │
│ ·日│                                           │
└────┴───────────────────────────────────────────┘
```

- **顶部栏**：固定定位，包含站点切换器（下拉选择当前站点）、通知铃铛（未读数badge）、用户菜单
- **侧边栏**：可折叠，根据当前角色动态显示菜单项，折叠状态仅显示图标
- **主内容区**：面包屑导航 + 页面标题 + 内容

#### 7.1.2 登录流程

```
访问后台 → 登录页(用户名+密码) → 验证 → [成功] → 选择站点(若多站点) → 仪表盘
                                      → [失败] → 错误提示 + 失败计数(5次锁定15分钟)
```

#### 7.1.3 内容管理流程

```
内容列表 → [新建] → 富文本编辑器 → 填写内容 → 保存草稿/提交审核
         → [编辑] → 富文本编辑器 → 修改内容 → 保存草稿/提交审核
         → [删除] → 确认弹窗 → 删除(进回收站)
```

#### 7.1.4 审核流程（P1）

```
待审列表 → 点击审核 → 查看内容 → [通过] → 内容状态变为"已发布"(或待下一级审核)
                             → [退稿] → 填写退稿原因 → 内容退回编辑 + 站内通知
```

#### 7.1.5 站点切换交互

- 顶部栏下拉选择站点
- 切换后侧边栏菜单刷新（可能因权限不同而变化）
- 切换后主内容区数据重新加载
- 切换动画过渡，当前站点名称醒目显示

### 8.3 文章状态机

```
草稿(draft) ──提交审核──→ 待审核(pending)
待审核(pending) ──审核通过──→ 已发布(published)
待审核(pending) ──审核退稿──→ 已退稿(rejected)
已退稿(rejected) ──修改后重新提交──→ 待审核(pending)
已发布(published) ──下线──→ 已下线(offline)
已下线(offline) ──重新发布──→ 已发布(published)
任意状态 ──删除──→ 回收站(trashed)
回收站(trashed) ──恢复──→ 草稿(draft)
回收站(trashed) ──永久删除──→ 物理删除
```
