## 1. 系统概述

### 1.1 系统简介

企业级CMS官网平台是一款面向教育集团/多法人机构的 Content Management System（内容管理系统），支持：
- **多站点独立运营**：每个法人站点数据、权限、模板完全隔离
- **多级权限管理**：超级管理员、站点管理员、编辑、审核员四级权限
- **完整内容工作流**：从采编 → 审核 → 发布的全流程管理
- **响应式前台展示**：自适应PC、平板、手机等多种设备

### 1.2 访问地址

| 环境 | 地址 | 说明 |
|------|------|------|
| **本地开发** | `http://localhost:5173/admin` | 需要启动前端开发服务器 |
| **前台官网** | `http://localhost:5173/` | 根据域名自动识别站点 |
| **生产环境** | 由部署配置决定 | 请咨询系统管理员 |

### 1.3 浏览器要求

- ✅ Chrome 90+
- ✅ Firefox 88+
- ✅ Safari 14+
- ✅ Edge 90+
- ❌ IE 11 及以下不支持

## 2. 登录与退出

### 2.1 登录步骤

1. 在浏览器访问后台地址（`http://localhost:5173/admin`）
2. 输入**用户名**和**密码**
3. 点击"登录"按钮
4. 登录成功后自动跳转到**仪表盘**

**默认账号**（开发环境）：
| 用户名 | 密码 | 角色 | 说明 |
|--------|------|------|------|
| `admin` | `admin123` | 超级管理员 | 拥有所有权限 |
| `dzyz_admin` | `admin123` | 站点管理员 | 德育中学站点管理员 |
| `cyjt_admin` | `admin123` | 站点管理员 | 春阳教育集团站点管理员 |

### 2.2 退出登录

1. 点击右上角**用户头像**
2. 选择"退出登录"
3. 系统清除 Token 并跳转到登录页

### 2.3 忘记密码

目前系统**不支持自助找回密码**，请联系系统管理员重置密码。

## 3. 用户管理

### 3.1 功能概述

用户管理模块用于管理后台系统的所有用户账号，包括：
- 新增用户
- 编辑用户信息
- 启用/禁用用户
- 重置用户密码
- 删除用户

**权限要求**：
- ✅ 超级管理员：可以管理所有用户
- ✅ 站点管理员：可以管理本站点用户
- ❌ 编辑/审核员：无权访问用户管理

### 3.2 新增用户

**操作步骤**：

1. 登录后台 → 左侧菜单 → **用户管理**
2. 点击右上角 **"新增用户"** 按钮
3. 填写用户信息：

| 字段 | 必填 | 说明 |
|------|------|------|
| 用户名 | ✅ | 唯一标识，创建后不可修改 |
| 密码 | ✅ | 最少6个字符 |
| 姓名 | ❌ | 用户真实姓名 |
| 邮箱 | ❌ | 用于通知和找回密码 |
| 电话 | ❌ | 联系电话 |
| 启用账号 | ✅ | 禁用后该用户无法登录 |

4. 点击 **"创建"** 按钮
5. 提示"用户创建成功"即完成

**注意事项**：
- 用户名一旦创建不可修改
- 密码最少6个字符
- 新增的用户默认**无角色**，需要到"站点管理 → 站点用户"中分配角色

### 3.3 编辑用户

**操作步骤**：

1. 在用户列表中找到目标用户
2. 点击该用户行的 **"编辑"** 按钮（铅笔图标）
3. 修改用户信息（**用户名不可修改**）
4. 点击 **"保存"** 按钮
5. 提示"用户更新成功"即完成

**可修改字段**：
- ✅ 姓名
- ✅ 邮箱
- ✅ 电话
- ✅ 启用状态

**不可修改字段**：
- ❌ 用户名（创建后不可修改）
- ❌ 密码（需要通过"重置密码"功能修改）

### 3.4 启用/禁用用户

**操作步骤**：

1. 在用户列表中找到目标用户
2. 点击 **"启用"** 或 **"禁用"** 按钮
3. 系统自动执行，提示"用户已禁用"或"用户已启用"

**效果**：
- **禁用**：该用户无法登录后台（Token 立即失效）
- **启用**：该用户恢复正常登录

### 3.5 重置密码

**操作步骤**：

1. 在用户列表中找到目标用户
2. 点击 **"重置密码"** 按钮（锁图标）
3. 在确认弹窗中点击 **"重置"**
4. 提示"密码重置成功"即完成
5. **新密码为 `123456`**（用户登录后请立即修改密码）

**注意事项**：
- 重置密码后，旧密码立即失效
- 新密码为默认密码 `123456`
- 建议用户在下次登录后立即修改密码

### 3.6 删除用户

**操作步骤**：

1. 在用户列表中找到目标用户
2. 点击 **"删除"** 按钮（垃圾桶图标）
3. 在确认弹窗中点击 **"删除"**
4. 提示"用户删除成功"即完成

**注意事项**：
- ⚠️ **删除操作不可恢复**
- ⚠️ 删除用户后，该用户创建的内容**不会被删除**（作者字段显示为"已删除用户"）
- ⚠️ 建议先"禁用"用户，确认不再需要后再删除

### 3.7 搜索用户

**操作步骤**：

1. 在用户管理页面顶部的搜索框中输入关键词
2. 系统实时筛选匹配的用户

**支持搜索的字段**：
- 用户名
- 姓名
- 邮箱

**示例**：
- 输入 `admin` → 显示所有用户名或姓名包含 `admin` 的用户
- 输入 `@example.com` → 显示所有邮箱包含 `@example.com` 的用户

### 3.8 分页与排序

- **分页**：底部显示总用户数和当前页，可以切换每页显示数量（10/20/50）
- **排序**：点击列表标题（如"用户名"）可以按该字段排序

## 4. 站点管理

### 4.1 功能概述

站点管理模块用于管理多法人站点，包括：
- 新增站点
- 编辑站点信息
- 切换当前站点
- 启用/停用站点
- 删除站点
- 管理站点用户

**权限要求**：
- ✅ 超级管理员：可以管理所有站点
- ❌ 站点管理员：只能管理自己所属的站点
- ❌ 编辑/审核员：无权访问站点管理

### 4.2 卡片式布局说明

从 **v2.1.0** 开始，站点管理采用**卡片式布局**（与原型 `admin.html` 对齐）：

- 每个站点显示为一个**卡片**
- 卡片显示：站点Logo、名称、ID、电话、地址、域名
- 支持 **3列网格布局**（宽屏下）
- 当前站点有**蓝色边框**标识

### 4.3 新增站点

**操作步骤**：

1. 登录后台 → 左侧菜单 → **站点管理**
2. 点击右上角 **"新增站点"** 按钮
3. 填写站点信息：

| 字段 | 必填 | 说明 |
|------|------|------|
| 站点名称（中文） | ✅ | 如"春阳教育集团" |
| 站点名称（英文） | ✅ | 如"site_001"（用作ID） |
| 域名 | ❌ | 如 `cyjt.localhost`（用于多域名访问） |
| 联系电话 | ❌ | 显示在前台底部 |
| 邮箱 | ❌ | 显示在前台底部 |
| Logo URL | ❌ | 站点Logo图片地址 |
| 地址 | ❌ | 显示在前台底部 |
| 主色调 | ❌ | 站点主题色（十六进制，如 `#3b82f6`） |
| 备案号 | ❌ | 显示在前台底部 |

4. 点击 **"创建"** 按钮
5. 提示"站点创建成功"即完成

### 4.4 切换当前站点

**操作步骤**：

1. 在站点列表中找到目标站点
2. 点击该站点卡片的 **"切换"** 按钮
3. 系统自动切换，提示"已切换到 XXX 站点"
4. 切换后，所有管理操作（文章、栏目、轮播图等）都针对该站点

**注意事项**：
- 切换站点只影响后台管理，不影响前台访问
- 前台访问根据 **域名自动识别** 站点（Scheme B）

### 4.5 管理站点用户

**操作步骤**：

1. 在站点列表中找到目标站点
2. 点击该站点卡片的 **"用户"** 按钮
3. 进入站点用户管理页面
4. 可以：
   - **添加用户**：将已有用户分配到该站点，并分配角色
   - **修改角色**：修改用户在该站点的角色
   - **移除用户**：将用户从该站点移除（不会删除用户账号）

**角色说明**：

| 角色 | 权限 |
|------|------|
| `SUPER_ADMIN` | 跨站点全局权限，无任何限制 |
| `SITE_ADMIN` | 管理本站点内容、人员、模板、配置 |
| `EDITOR` | 内容采编、提交审核、管理个人素材 |
| `REVIEWER` | 审核内容、退稿、标记问题内容 |
| `VIEWER` | 仅可查看，无权修改 |

## 5. 内容管理

### 5.1 文章管理

**功能概述**：
- 新增文章
- 编辑文章
- 发布/取消发布文章
- 删除文章
- 按栏目分类管理

**操作步骤**（简述）：
1. 左侧菜单 → **内容管理** → **文章管理**
2. 点击"新增文章" → 填写标题、内容、封面图、栏目等 → 保存
3. 可以"保存草稿"或"提交审核"（如果开启了审核流程）
4. 审核通过后，点击"发布"即可在前台显示

### 5.2 栏目管理

**功能概述**：
- 管理多级栏目树
- 支持无限级嵌套
- 控制前台导航结构

**操作步骤**（简述）：
1. 左侧菜单 → **内容管理** → **栏目管理**
2. 可以新增顶级栏目，或在现有栏目下新增子栏目
3. 设置栏目名称、排序、是否显示在导航等

### 5.3 轮播图管理

**功能概述**：
- 管理前台首页轮播图
- 支持设置标题、副标题、链接、排序

**从 v2.1.0 新增功能**：
- ✅ 新增 **"副标题"** 字段
- ✅ 改为 **卡片式布局**

## 6. 常见问题

### 6.1 登录相关

**Q：忘记密码怎么办？**  
A：请联系系统管理员重置密码。重置后密码为 `123456`。

**Q：登录后提示"无权限访问"？**  
A：你的账号可能没有分配角色。请联系站点管理员到"站点管理 → 站点用户"中为你分配角色。

### 6.2 用户管理相关

**Q：新增用户后无法登录？**  
A：检查：
1. 用户是否被分配了角色（站点管理 → 站点用户）
2. 用户账号是否处于"启用"状态
3. 密码是否输入正确（区分大小写）

**Q：如何批量导入用户？**  
A：当前版本不支持批量导入，请逐个创建用户。后续版本会考虑增加此功能。

### 6.3 站点管理相关

**Q：切换站点后数据没有变化？**  
A：尝试刷新浏览器（F5 或 Cmd+R）。如果仍无变化，请清除浏览器缓存后重试。

**Q：删除了站点，数据还能恢复吗？**  
A：❌ 删除站点会同时删除该站点的所有内容（文章、栏目、轮播图等），且不可恢复。建议先"停用"站点，确认不再需要后再删除。

### 6.4 内容管理相关

**Q：文章发布后前台不显示？**  
A：检查：
1. 文章是否处于"已发布"状态
2. 文章所属栏目是否处于"启用"状态
3. 前台是否访问了正确的域名（对应正确的站点）

**Q：上传的图片无法显示？**  
A：检查图片URL是否正确，以及图片文件是否存在于服务器上。
