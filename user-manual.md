# 官网CMS系统 - 用户手册

**版本：** v2.0  
**更新日期：** 2026-05-25  

---

## 1. 系统简介

本系统是一个**综合性官网内容管理系统（CMS）**，支持多法人/多站点管理。通过后台灵活配置，即可快速生成不同机构的官网页面。

### 1.1 核心特性
- **多站点管理**：支持多个法人/机构，每个站点有独立配置
- **零代码建站**：后台配置即可生成完整官网
- **响应式设计**：自动适配PC端和移动端
- **主题可定制**：每个站点可设置独立的主色调/副色调

### 1.2 系统组成

| 文件 | 说明 |
|------|------|
| `index.html` | 前台官网展示页 |
| `admin.html` | 后台管理系统 |
| `cms-data.js` | 数据初始化层 |
| `test-report.md` | 测试报告 |

## 2. 快速开始

### 2.1 启动系统
1. 将 `official-website/` 文件夹放在任意Web服务器目录下

## 3. 后台管理操作指南

### 3.1 登录系统
1. 打开 `admin.html`
2. 输入用户名和密码
3. 点击"登录"按钮

### 3.2 仪表盘
登录后默认进入仪表盘，展示当前站点的数据统计：
- 文章总数
- 访问量
- 站点数量
- 用户数

### 3.3 站点管理（多法人核心功能）

**切换站点：**
1. 点击左侧菜单"站点管理"
2. 在站点卡片上点击"切换"按钮
3. 切换后，所有配置页面自动切换到该站点的数据

**新增站点：**
1. 点击"新增站点"按钮
2. 填写站点名称、英文名、联系电话、邮箱、地址等
3. 选择主色调和副色调
4. 填写网站简介
5. 点击"创建"

**删除站点：**
1. 在站点卡片上点击"删除"
2. 确认删除操作
3. 系统至少保留一个站点

> 💡 **重要**：切换站点后，前台 `index.html` 会自动读取当前激活站点的配置并渲染对应的官网页面。

### 3.4 基本配置
配置当前站点的基本信息：
- **站点名称**：中文全称
- **英文名称**：英文全称
- **Logo URL**：Logo图片地址
- **联系电话**：显示在顶部和底部
- **邮箱**：显示在顶部和底部
- **地址**：显示在底部
- **备案号**：显示在底部
- **公安备案**：显示在底部
- **主色调**：网站主色（使用颜色选择器）
- **副色调**：按钮/装饰色（使用颜色选择器）
- **网站简介**：首页关于我们模块的内容

点击"保存配置"后，数据立即写入 localStorage，前台刷新即可看到变化。

### 3.5 导航管理
管理网站的顶部导航菜单：

**一级菜单操作：**
- 点击"添加一级菜单"新增
- 直接修改名称和链接
- 点击删除图标移除

**二级菜单操作：**
- 在一级菜单下方点击"添加子菜单"
- 修改子菜单名称和链接
- 点击 × 删除子菜单

### 3.6 内容管理
管理网站的文章/新闻：

**新增文章：**
1. 点击"新增文章"
2. 填写标题、分类、日期、封面图URL、摘要
3. 选择状态：发布/草稿
4. 点击"保存"

**编辑文章：**
- 点击文章行的"编辑"按钮
- 修改字段后保存

**发布/取消发布：**
- 点击"发布"或"取消发布"切换文章状态

**删除文章：**
- 点击"删除"按钮，确认后删除

### 3.7 轮播图管理
管理首页的轮播Banner：

1. 点击"添加"新增轮播图
2. 填写标题、副标题、图片URL、链接
3. 已有轮播图可直接在卡片内修改
4. 点击"删除"移除

> 💡 推荐图片尺寸：1400×500像素

### 3.8 快捷入口管理
管理首页的4个快捷入口按钮：

1. 点击"添加"新增
2. 设置名称、图标（Font Awesome class）、链接、颜色
3. 常用图标class：`fa-headset`、`fa-pen-to-square`、`fa-download`、`fa-envelope`

> 💡 更多图标参考：https://fontawesome.com/icons

### 3.9 友情链接管理
管理底部友情链接区域：
- 点击"添加"新增
- 设置名称和URL
- 直接在表格内修改
- 点击"删除"移除

### 3.10 领导介绍管理
管理首页"领导班子"模块：

1. 点击"添加"新增
2. 填写姓名、职务、照片URL、简介
3. 在卡片内可直接修改
4. 点击"删除"移除

### 3.11 师资管理
管理首页"师资队伍"模块：

1. 点击"添加"新增教师
2. 填写姓名、学科、职称、从教年限、照片URL、简介
3. 在卡片内可直接修改
4. 点击"删除"移除

### 3.12 用户管理
管理系统登录用户（全局，跨站点共享）：

- 查看用户列表：用户名、角色、状态、最后登录时间
- 启用/禁用用户：点击"启用"或"禁用"切换
- 新增用户：填写用户名、密码、角色
- 删除用户：点击"删除"确认

## 4. 前台展示说明

### 4.1 页面结构（从上到下）
1. **顶部工具栏** - 电话、邮箱、管理登录入口
2. **Logo区域** - 机构名称（中英文）、搜索框
3. **主导航栏** - 一级菜单 + 二级下拉菜单
4. **轮播Banner** - 全宽大图轮播，5秒自动切换
5. **快捷入口** - 4个图标按钮
6. **学校要闻 + 通知公告** - 左右分栏
7. **学校简介** - 左图右文 + 数据统计
8. **师资队伍** - 教师卡片展示
9. **领导班子** - 领导卡片展示
10. **友情链接** - 链接列表
11. **底部版权** - 联系方式、备案信息

### 4.2 数据加载机制
- 前台启动时调用 `initCMSData()` 初始化数据
- 通过 `getCurrentSiteData()` 获取当前激活站点的完整配置
- 所有模块动态渲染，无硬编码内容
- 如果localStorage中没有数据，自动使用默认数据

### 4.3 主题切换
- 每个站点有独立的 `primaryColor`（主色调）和 `secondaryColor`（副色调）
- 通过CSS变量 `--primary` 和 `--secondary` 实现
- 切换站点后，主题色自动更新

## 5. 多站点/多法人使用流程

### 5.1 创建新站点
1. 登录后台 → 站点管理 → 新增站点
2. 填写机构名称、主色调等基本信息
3. 切换到新站点
4. 依次配置：导航菜单 → 轮播图 → 快捷入口 → 内容 → 师资 → 领导 → 友情链接
5. 打开前台 `index.html` 查看效果

### 5.2 管理多个站点
- 在站点管理页面可随时切换当前站点
- 切换后所有配置自动加载该站点的数据
- 各站点数据完全独立，互不影响

## 6. 技术说明

### 6.1 数据存储
- 所有数据存储在浏览器 `localStorage` 中
- 存储key：`cms_site_data`
- 数据结构：`{ currentSiteId, sites: [...], users: [...] }`
- 每个站点是sites数组中的一个独立对象

### 6.2 核心API（cms-data.js）

| 函数 | 说明 |
|------|------|
| `initCMSData()` | 初始化数据（首次加载写入默认值） |
| `getCMSData()` | 获取全部CMS数据 |
| `saveCMSData(data)` | 保存全部CMS数据 |
| `getCurrentSiteId()` | 获取当前激活站点ID |
| `setCurrentSiteId(id)` | 设置当前激活站点ID |
| `getCurrentSiteData()` | 获取当前激活站点的完整数据 |

### 6.3 站点数据结构

```
{
  id: string,              // 站点唯一标识
  name: string,            // 中文名称
  nameEn: string,          // 英文名称
  primaryColor: string,    // 主色调 (#hex)
  secondaryColor: string,  // 副色调 (#hex)
  logo: string,            // Logo图片URL
  phone: string,           // 联系电话
  email: string,           // 邮箱
  address: string,         // 地址
  icp: string,             // 备案号
  police: string,          // 公安备案号
  description: string,     // 网站简介
  navItems: [...],         // 导航菜单
  banners: [...],          // 轮播图
  quickLinks: [...],       // 快捷入口
  news: [...],             // 文章列表
  announcements: [...],    // 公告列表
  teachers: [...],         // 师资队伍
  leaders: [...],          // 领导介绍
  friendLinks: [...],      // 友情链接
  stats: {...}             // 统计数据
}
```

## 7. 常见问题

### Q1：前台页面空白怎么办？
**A：** 清除浏览器localStorage后刷新页面：
- 打开浏览器开发者工具（F12）→ Application → Local Storage → 删除 `cms_site_data`
- 刷新页面，系统会自动重新初始化默认数据

### Q2：切换站点后前台没有变化？
**A：** 前台页面需要手动刷新（F5）才能读取最新的站点配置。

### Q3：图片显示不了？
**A：** 图片使用的是外部CDN（unsplash），需要网络连接。可以替换为自己的图片URL。

### Q4：如何修改默认数据？
**A：** 编辑 `cms-data.js` 中的 `DEFAULT_DATA` 对象，修改后清除localStorage刷新即可。

### Q5：如何对接真实后端？
**A：** 将 `cms-data.js` 中的 localStorage 读写函数替换为API调用：
- `getCMSData()` → `fetch('/api/cms-data')`
- `saveCMSData(data)` → `fetch('/api/cms-data', { method: 'POST', body: data })`

### Q6：忘记密码怎么办？
**A：** 清除浏览器localStorage后刷新，默认账号 admin/admin123 会恢复。

### Q7：如何部署到线上？
**A：** 将3个文件上传到任意静态文件服务器（Nginx/Apache/GitHub Pages/云存储），无需额外配置。
